## 九、核心差异总结

### 上游的定位

> 一个**学术研究导向**的多 Agent 金融分析框架。核心创新在论文（arXiv 2412.20138），代码是概念验证。目标用户是研究者，通过 CLI 做单次分析调用。

### 本 Fork 的定位

> 一个**产品化导向**的 A 股智能分析平台。在继承上游多 Agent 架构的基础上，增加了知识管理、智能调度、持续采集、多渠道交互、多用户支持——使其能从"单次分析工具"进化为"24/7 运行的类券商研究所"。

### 三个根本差异

```
1. 数据层
   上游: yfinance（美股）→ 够用
   Fork:  AkShare + 国信证券（A 股）→ 必须有

2. 运行模式
   上游: CLI 单次触发 → 分析 → 输出 → 结束
   Fork:  后台采集 → KB 积累 → API 触发/定时 → 分析 → 推送 → 归档

3. 知识管理
   上游: TradingMemoryLog（反思用）→ 被动
   Fork:  KB + Archive + ContextAssembler + 置信度标签 → 主动
```

### 量化差异
