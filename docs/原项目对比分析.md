## 二、架构对比

### 2.1 整体架构

| 原项目 | 本项目 |
|------|------|
| 单体 CLI 应用 | 双层智能系统 |
| LangGraph 静态图（固定 11 节点） | LangGraph 静态图 + 动态图（V1.2 Planner） |
| 单次同步执行 | 后台采集 + 事件驱动 + HTTP API |
| 无知识管理 | KB + Freshness + ContextAssembler |
| 无调度机制 | Scheduler（interval + cron） |
| 无多用户概念 | 按 user_id 数据隔离 |

```
原项目:  CLI → LangGraph 固定管道 → 终端输出

本项目:  OpenClaw/HTTP API → Planner → DynamicGraphBuilder
              ↑                      ↑
         后台 Collector → KB ────────┘
              ↓
         通知推送（飞书/微信/Server酱）
```

### 2.2 Agent 管线

两项目共享相同的 LangGraph 辩论核心拓扑：

```
分析师链 → 投资辩论 (Bull↔Bear) → Research Manager → Trader
         → 风控辩论 (Aggressive↔Conservative↔Neutral) → Portfolio Manager → END
```

**本项目增强**：
- 分析师串行 → 待改造为并行（FIX-1）
- 辩论路由 `startswith("Bull")` → 待改造为枚举（FIX-2）
- 新增 ContextAssembler（图执行前注入历史知识）
- 新增 Market Context 注入（大盘环境 + 非交易日降级）
- 新增 A 股约束注入（涨跌停/T+1）
- 新增知识上下文注入（CONFIRMED/SINGLE/CONFLICTING/STALE 置信度标签）

## 十、关键设计差异总结

| 维度 | 原项目设计哲学 | 本项目设计哲学 |
|------|------|------|
| **定位** | 学术研究框架 | 生产级交易平台 |
| **交互方式** | "给定股票+日期，输出建议" | "模拟券商研究所，持续运行" |
| **数据策略** | 实时拉取（每次分析从零开始） | 预采集 + 缓存 + KB 复用（降 55% 成本） |
| **Agent 调度** | 全量 11 Agent 始终运行 | 按模板/意图按需调度 |
| **知识积累** | 单文件 Markdown 反思 | 五层架构（计划中）+ 置信度标签 |
| **市场适配** | 美股通用 | A 股深适配（涨跌停/T+1/交易日历/沪深300基准） |
| **部署模式** | 本地 CLI 单次运行 | Docker + HTTP API + 定时调度 + 多渠道推送 |
