## FIX-1：分析师并行化

**优先级**：P0 | **预估工时**：1.5 天 | **影响文件**：`graph/setup.py`

### 问题

当前 `GraphSetup.setup_graph()` 第 112-134 行将四个分析师硬编码为串行链：

```python
# 当前代码 (setup.py:112-134)
for i, analyst_type in enumerate(selected_analysts):
    # ...
    if i < len(selected_analysts) - 1:
        workflow.add_edge(current_clear, next_analyst)  # 串行
    else:
        workflow.add_edge(current_clear, "Bull Researcher")
```

四个分析师读取完全独立的数据源（市场数据、社交媒体、新闻、财务），彼此零依赖。串行导致总耗时 = 四者之和（~270s），而非四者最大值（~90s）。

### 修复方案

使用 LangGraph 的 `Send` API 实现扇出-汇聚（fan-out / fan-in）模式。

#### Step 1：创建并行扇出入口节点

在 `setup.py` 中新增 `fan_out_analysts` 节点：

```python
# 新增: graph/setup.py

def create_fan_out_analysts(selected_analysts: list[str]):
    """扇出节点：为每个选中的分析师生成一个 Send 到对应节点。"""
    def fan_out(state: AgentState):
        sends = []
        for analyst_type in selected_analysts:
            analyst_name = f"{analyst_type.capitalize()} Analyst"
            sends.append(Send(analyst_name, state))
        return sends
    return fan_out
```

#### Step 2：创建汇聚节点

```python
# 新增: graph/setup.py

def create_merge_analyst_reports(state: AgentState) -> AgentState:
    """汇聚节点：四个分析师完成后的聚合逻辑。

    此时 state 中的 market_report/sentiment_report/news_report/
    fundamentals_report 已被各 Analyst 节点写入。
    此节点不需要做任何额外的状态修改，仅作为同步屏障。
    """
    # 验证所有报告非空
    reports = {
        "market": state.get("market_report", ""),
        "sentiment": state.get("sentiment_report", ""),
        "news": state.get("news_report", ""),
        "fundamentals": state.get("fundamentals_report", ""),
    }
    missing = [k for k, v in reports.items() if not v]
    if missing:
        logger.warning("Analyst reports missing for: %s", missing)
    return state
```

#### Step 3：修改图拓扑


## FIX-2：辩论路由枚举化

**优先级**：P0 | **预估工时**：1 天 | **影响文件**：`graph/conditional_logic.py`、`agents/researchers/bull_researcher.py`、`agents/researchers/bear_researcher.py`

当前投资辩论路由依赖 `current_response.startswith("Bull")` 字符串匹配：

```python
    debate = state["investment_debate_state"]
```

## FIX-3：V1.2 动态图支持检查点

**优先级**：P0 | **预估工时**：1.5 天 | **影响文件**：`graph/executor.py`、`graph/checkpointer.py`

### 问题

V1.2 的 `GraphExecutor.execute()` 不支持检查点，而它正是 `POST /analyze` API 的入口。这意味着 API 调用中途崩溃后：
- 已消耗的 LLM tokens 全部浪费
- 调用方得不到任何恢复机制
- 必须重新触发完整分析

### 修复方案

在 `GraphExecutor` 中集成与 V1.0 相同的检查点机制。

#### Step 2：execute() 中集成检查点

```python
# graph/executor.py — execute() 方法改造

def execute(self, plan, trigger, context):
    plan = self._normalize_plan(plan)

    # 构建唯一标识（与 V1.0 的 thread_id 逻辑一致）
    task_id = f"{context.ticker or 'market'}:{context.user_id}:{trigger.type}"
    task_hash = hashlib.sha256(task_id.encode()).hexdigest()[:16]

    init_state = self._build_init_state(plan, trigger, context)
    builder = DynamicGraphBuilder(self.quick_llm, self.deep_llm, self.tool_nodes)
    workflow = builder.build(plan)

    checkpoint_saver = None
    ctx = None
    try:
        if self.enable_checkpoint:
            from .checkpointer import get_checkpointer
            ctx = get_checkpointer(
                self.config.get("data_cache_dir", "~/.tradingagents/cache"),
                task_hash
            )
            checkpoint_saver = ctx.__enter__()
            compiled = workflow.compile(checkpointer=checkpoint_saver)
        else:
            compiled = workflow.compile()

        result = compiled.invoke(
            init_state,
            {"configurable": {"thread_id": task_hash}}
        )

        # 成功完成后清除检查点
        if self.enable_checkpoint and ctx:
            from .checkpointer import clear_checkpoint
            clear_checkpoint(
                self.config.get("data_cache_dir", "~/.tradingagents/cache"),
                task_hash
            )

    except Exception as e:
        logger.error("Graph execution failed for %s: %s", task_hash, e)
        # 保留检查点，下次调用自动恢复
        raise
    finally:
        if ctx:
            ctx.__exit__(None, None, None)

    return self._build_result(result, plan)
```

## FIX-4：deep_llm Fallback 机制

**优先级**：P0 | **预估工时**：0.5 天 | **影响文件**：`graph/trading_graph.py`、`graph/executor.py`

### 问题

Research Manager 和 Portfolio Manager 都依赖 `deep_llm`。如果 deep_llm 提供商宕机，两个节点全部卡死，整个分析失败。

### 修复方案

在 LLM 客户端层实现自动降级。

```python
# llm_clients/ — 新增 resilient_llm.py

from typing import Any, Optional
import logging

logger = logging.getLogger(__name__)

class ResilientLLM:
    """LLM 包装器，支持自动 fallback。

    当 primary LLM 调用失败时（超时/5xx/配额耗尽），
    自动降级到 fallback LLM 重试。
    """

    def __init__(
        self,
        primary: Any,          # deep_llm
        fallback: Any = None,  # quick_llm（可选）
        max_retries: int = 2,
        retry_delay: float = 3.0,
    ):
        self.primary = primary
        self.fallback = fallback
        self.max_retries = max_retries
        self.retry_delay = retry_delay
        self._degraded = False

    @property
    def is_degraded(self) -> bool:
        """当前是否已降级。"""
        return self._degraded

    def invoke(self, prompt, **kwargs):
        """调用 LLM，失败时自动 fallback。"""
        errors = []

        # 尝试 primary（含重试）
        for attempt in range(self.max_retries):
            try:
                result = self.primary.invoke(prompt, **kwargs)
                self._degraded = False
                return result
            except Exception as e:
                errors.append(str(e))
                if attempt < self.max_retries - 1:
                    logger.warning(
                        "primary LLM attempt %d/%d failed: %s",
                        attempt + 1, self.max_retries, e
                    )
                    import time
                    time.sleep(self.retry_delay)

        # Primary 全部失败 → 尝试 fallback
        if self.fallback:
            logger.warning(
                "primary LLM exhausted (%d attempts). "
                "Falling back to secondary LLM. Errors: %s",
                self.max_retries, errors
            )
            try:
                result = self.fallback.invoke(prompt, **kwargs)
                self._degraded = True
                # 在结果中标记降级状态（Agent 可在输出中标注）
                return result
            except Exception as e:
                logger.error("fallback LLM also failed: %s", e)
                raise RuntimeError(
                    f"Both primary and fallback LLMs failed. "
                    f"Primary errors: {errors}; Fallback error: {e}"
                )

        raise RuntimeError(
            f"Primary LLM failed after {self.max_retries} attempts "
            f"and no fallback configured. Errors: {errors}"
        )

    def with_structured_output(self, schema, **kwargs):
        """结构化输出调用（同样支持 fallback）。"""
        # 先尝试 primary 的结构化输出
        try:
            return self.primary.with_structured_output(schema, **kwargs)
        except Exception:
            if self.fallback:
                logger.warning("primary structured_output failed, using fallback")
                return self.fallback.with_structured_output(schema, **kwargs)
            raise
```

## FIX-5：辩论深度默认值提升 + 质量度量

**优先级**：P1 | **预估工时**：1.5 天 | **影响文件**：`default_config.py`、`graph/conditional_logic.py`、新增 `graph/debate_quality.py`

### 问题

默认 `max_debate_rounds=1` 导致 Bull 首轮 → Bear 反驳 → 立即裁判。Bull 没有机会回应 Bear 的指控。这不是辩论，是各说一句话的"形式主义"。

同时，系统不检查辩论质量——新证据是否引入、论据是否重复、观点是否被实际挑战等。

### 修复方案

#### Step 1：提升默认值

```python
# default_config.py — 修改默认配置

DEFAULT_CONFIG = {
    # ...
    "max_debate_rounds": 2,          # 1 → 2（Bull→Bear→Bull→Bear→RM）
    "max_risk_discuss_rounds": 2,    # 1 → 2（Agr→Cons→Neut→Agr→Cons→Neut→PM）
    # ...
}
```

**辩论深度变化**：

| 配置 | 发言序列 | 实质对抗 |
|:---:|------|:---:|
| 旧: `max=1` | Bull → Bear → RM | Bull 没机会反驳 Bear |
| 新: `max=2` | Bull → Bear → Bull → Bear → RM | Bull 可以反驳 Bear 的首轮指控 |

#### Step 3：集成到条件路由

```python
# conditional_logic.py — 增强 should_continue_debate

def should_continue_debate(self, state: AgentState) -> str:
    debate = state["investment_debate_state"]

    # 硬上限保护（不变）
    if debate["count"] >= 2 * self.max_debate_rounds:
        return "Research Manager"

    # 质量检查（新增）：连续 2 轮低质量 → 提前终止
    if debate["count"] >= 4:  # 至少跑完 2 个完整来回
        quality = DebateQualityTracker.should_continue_with_quality(
            state, self.max_debate_rounds
        )
        if quality == "terminate":
            logger.info("Debate terminated early due to quality degradation")
            return "Research Manager"

    # 枚举路由（FIX-2 修复后）
```

## FIX-6：KB 覆盖率时效加权

**优先级**：P1 | **预估工时**：0.5 天 | **影响文件**：`kb/knowledge_base.py`

### 问题

当前 `_calculate_coverage()` 只看数据是否存在，不看数据的时效性：

```python
# knowledge_base.py:144-157 — 当前实现
def _calculate_coverage(self, results):
    total = len(self.collections)
    available = sum(1 for v in results.values() if v is not None)
    return available / total if total > 0 else 0.0
```

导致：90 分钟前的"市场快照"虽然标记为 FRESH（仍在 30min TTL 内），但在市场剧烈波动时实际已经过时。Planner 却基于此数据跳过分析。

### 修复方案

为覆盖率计算加入时效衰减因子。

```python
# knowledge_base.py — 重写 _calculate_coverage

import time

# 时效衰减配置（秒）
DECAY_CONFIG = {
    "market_snapshot":     {"half_life": 600},     # 10分钟半衰期
    "sentiment_report":    {"half_life": 300},     # 5分钟
    "announcement_brief":  {"half_life": 1800},    # 30分钟
    "policy_brief":        {"half_life": 3600},    # 60分钟
    "stock_snapshot":      {"half_life": 1800},    # 30分钟
}

def _calculate_coverage(self, results: dict) -> tuple[float, dict]:
    """计算 KB 覆盖率（含时效衰减）。

    Returns:
        (weighted_coverage, detail)
        detail = {
            "raw_coverage": float,      # 原始覆盖率
            "weighted_coverage": float, # 时效加权覆盖率
            "stale_items": list[str],   # 已过时效的数据项
        }
    """
    total_weight = 0.0
    available_weight = 0.0
    stale_items = []

    for collection_name, data in results.items():
        decay_cfg = DECAY_CONFIG.get(collection_name, {"half_life": 1800})
        base_weight = 1.0  # 每类数据基础权重相同

        if data is not None:
            # 获取数据时间戳
            timestamp = data.get("_ts", 0)
            age_seconds = time.time() - timestamp

            # 指数衰减: w = base × 0.5^(age / half_life)
            decay_factor = 0.5 ** (age_seconds / decay_cfg["half_life"])
            effective_weight = base_weight * decay_factor
            available_weight += effective_weight

            if decay_factor < 0.25:  # 衰减超过 75% → 标记为过时
                stale_items.append(collection_name)

        total_weight += base_weight

    raw_coverage = available_weight / total_weight if total_weight > 0 else 0.0

    # 若有 stale 项，额外扣分（奖励及时采集）
    stale_penalty = len(stale_items) * 0.1
    weighted_coverage = max(0.0, raw_coverage - stale_penalty)

    return weighted_coverage, {
        "raw_coverage": raw_coverage,
        "weighted_coverage": weighted_coverage,
        "stale_items": stale_items,
    }
```

```python
# 新建: graph/context_manager.py

class ContextWindowManager:
    """上下文窗口管理器。

    三级策略管理 Agent 的上下文注入：
    Level 1: Token 预算监控 — 实时追踪总 token 数
    Level 2: 结构化摘要 — 超过预算时压缩而非截断
    Level 3: 硬截断 — 紧急情况下的最后防线
    """

    DEBATE_TOKEN_BUDGET = 4000    # 辩论历史最大 token 数
    REPORT_TOKEN_BUDGET = 8000    # 分析师报告最大 token 数

    @staticmethod
    def estimate_tokens(text: str) -> int:
        """粗略 token 估算（4 字符 ≈ 1 token）。"""
        return len(text) // 4

    @staticmethod
    def summarize_if_needed(
        history: str,
        reports: dict[str, str],
        max_tokens: int,
        quick_llm,
    ) -> str:
        """当上下文超过 token 预算时，用 LLM 生成结构化摘要。

        摘要格式:
        ```
        ## 分析师报告摘要
        - Market: [核心结论，1-2句]
        - Fundamentals: [核心结论，1-2句]
        - News: [核心结论，1-2句]
        - Social: [核心结论，1-2句]

        ## 辩论历史摘要（最近2轮）
        - Bull [轮次1]: [核心论点 + 关键证据]
        - Bear [轮次1]: [核心论点 + 关键证据]
        - Bull [轮次2]: [核心论点 + 关键证据]
        - Bear [轮次2]: [核心论点 + 关键证据]
        ```
        """
        total_tokens = ContextWindowManager.estimate_tokens(history)
        for report in reports.values():
            total_tokens += ContextWindowManager.estimate_tokens(report)

        if total_tokens <= max_tokens:
            return history  # 无需压缩

        # 超过预算 → 生成摘要
        summary_prompt = f"""
        将以下分析报告和辩论历史压缩为结构化摘要。
        保留：具体数字、关键指标、核心论点、证据引用。
        丢弃：重复表述、过度修饰、冗余分析。
        目标长度：不超过 {max_tokens // 4} 字符。

        报告:
        {reports}

        历史:
        {history[-3000:]}  # 只摘要最近内容
        """

        try:
            result = quick_llm.invoke(summary_prompt)
            content = result.content if hasattr(result, 'content') else str(result)
            return content[:max_tokens * 4]  # 字符上限兜底
        except Exception as e:
            logger.warning("Context summarization failed: %s", e)
            # 回退到简单截断（保留最近内容）
            return history[-max_tokens * 4:]

    @staticmethod
    def inject_context(
        state: AgentState,
        agent_type: str,  # "bull" | "bear" | "aggressive" | "conservative" | "neutral"
        quick_llm,
    ) -> dict:
        """为指定 Agent 准备优化后的上下文。

        Returns:
            {
                "reports_summary": str,   # 分析师报告（可能已压缩）
                "debate_history": str,    # 辩论历史（可能已压缩）
                "opponent_last": str,     # 对手最新发言（始终完整保留）
                "token_usage": int,       # 上下文 token 估算
                "compression_applied": bool,
            }
        """
        pass
```

## FIX-8：工具调用死循环检测

**优先级**：P2 | **预估工时**：0.5 天 | **影响文件**：`graph/conditional_logic.py`

### 问题

分析师节点只有一个条件判断：`if last_message.tool_calls → 继续工具循环`。如果 LLM 连续对同一工具同一参数重复调用（幻觉或数据返回异常），会一直循环到全局 `recursion_limit`。

### 修复方案

在条件路由中增加工具调用循环检测。

```python
# conditional_logic.py — 增强 should_continue_* 系列方法

class ConditionalLogic:
    def __init__(self, max_debate_rounds=1, max_risk_discuss_rounds=1):
        self.max_debate_rounds = max_debate_rounds
        self.max_risk_discuss_rounds = max_risk_discuss_rounds
        # 工具调用限制
        self.max_tool_calls_per_analyst = 12      # 每个分析师最多 12 次工具调用
        self.max_repeat_calls = 3                 # 同一工具+参数最多重复 3 次

    def _detect_tool_loop(self, state: AgentState, analyst_type: str) -> tuple[bool, str]:
        """检测工具调用死循环。

        检查三种退化模式：
        1. 连续调用同一工具 + 相同参数（死循环）
        2. 总工具调用次数超限（耗尽）
        3. 工具调用无进展（交替两个工具但没有新信息）
        """
        messages = state["messages"]
        tool_calls = [
            msg for msg in messages[-20:]  # 只看最近 20 条
            if hasattr(msg, 'tool_calls') and msg.tool_calls
        ]

        # 检查 1: 连续重复
        last_calls = []
        for msg in tool_calls[-5:]:  # 最近 5 次工具调用
            for tc in msg.tool_calls:
                call_key = f"{tc.get('name', '')}:{str(tc.get('args', {}))}"
                last_calls.append(call_key)

        # 如果最近 5 次中有 ≥ 3 次完全相同 → 死循环
        from collections import Counter
        counter = Counter(last_calls)
        most_common_count = counter.most_common(1)[0][1] if counter else 0
        if most_common_count >= self.max_repeat_calls:
            logger.warning(
                "Tool loop detected for %s: %d repeat calls of '%s'",
                analyst_type, most_common_count,
                counter.most_common(1)[0][0]
            )
            return True, "repeat_detected"

        # 检查 2: 总调用次数超限
        tool_msg_count = sum(
            1 for msg in messages
            if hasattr(msg, 'tool_calls') and msg.tool_calls
        )
        if tool_msg_count >= self.max_tool_calls_per_analyst:
            logger.warning(
                "Tool call limit exceeded for %s: %d calls",
                analyst_type, tool_msg_count
            )
            return True, "limit_exceeded"

        return False, "ok"

    def should_continue_market(self, state: AgentState):
        messages = state["messages"]
        last_message = messages[-1]

        if last_message.tool_calls:
            # 死循环检测
            is_loop, reason = self._detect_tool_loop(state, "market")
            if is_loop:
                logger.warning("Breaking market analyst tool loop: %s", reason)
                # 注入提示：停止工具调用，基于已有数据生成报告
                state["messages"].append(
                    HumanMessage(content=(
                        "工具调用已终止（原因：{reason}）。"
                        "请基于已获取的数据生成你的分析报告。"
                        "如果数据不足，请标注局限性并继续。"
                    ).format(reason=reason))
                )
                return "continue"  # 或路由到消息清除

            return "tools_market"

        return "continue"  # 而非 "Msg Clear Market"（FIX-1 并行化后）
```

### 预期效果

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| LLM 重复调用同一工具 | 持续到 recursion_limit | 3 次重复 → 强制终止并提示 |
| 工具调用疯狂 | 无上限 | 12 次上限 → 强制终止 |
| 浪费控制 | 无 | 每次最多 ~$0.05 额外成本 |

## FIX-9：并发安全保护

**优先级**：P2 | **预估工时**：0.5 天 | **影响文件**：`portfolio/portfolio_manager.py`、`agents/utils/position_state.py`

### 问题

`PositionStateManager` 和 `PortfolioManager` 使用文件系统 YAML 读写，无锁机制。多次并发分析同一 ticker 可能导致持仓状态撕裂写入。

### 修复方案

使用文件锁（`fcntl` 或 `filelock`）保护持仓文件。

```python
# agents/utils/position_state.py — 增加文件锁

import fcntl
import time
from contextlib import contextmanager

class PositionStateManager:
    LOCK_TIMEOUT = 5.0  # 锁等待超时（秒）

    @contextmanager
    def _lock_position_file(self, ticker: str):
        """获取持仓文件的排他锁。"""
        lock_path = Path(self.data_dir) / "positions" / f"{ticker}.lock"
        lock_path.parent.mkdir(parents=True, exist_ok=True)

        lock_file = open(lock_path, 'w')
        start = time.time()

        while True:
            try:
                fcntl.flock(lock_file.fileno(), fcntl.LOCK_EX | fcntl.LOCK_NB)
                break
            except BlockingIOError:
                if time.time() - start > self.LOCK_TIMEOUT:
                    lock_file.close()
                    raise TimeoutError(
                        f"Could not acquire lock for {ticker} "
                        f"within {self.LOCK_TIMEOUT}s"
                    )
                time.sleep(0.1)

        try:
            yield
        finally:
            fcntl.flock(lock_file.fileno(), fcntl.LOCK_UN)
            lock_file.close()

    def save(self, ticker: str, cost_price: float,
             quantity: int, opened_date: str = "") -> None:
        with self._lock_position_file(ticker):
            # 现有写入逻辑（不变）
            ...

    def load(self, ticker: str) -> Optional[dict]:
        with self._lock_position_file(ticker):
            # 现有读取逻辑（不变）
            ...
```

### 问题

当一次分析返回 "Hold" 时，无法追溯到是哪个 Agent 在哪个环节导致了这个决策。

### 修复方案

实现轻量级因果追踪——在关键决策点记录 `(决策, 依据, 来源)` 三元组。

```python
# 新建: graph/causal_tracer.py

import json
from datetime import datetime
from typing import Any

class CausalTracer:
    """因果追踪器——记录决策链上的每个关键节点。

    追踪格式:
    {
        "trace_id": "xxx",
        "timestamp": "2026-05-28T10:30:00",
        "decision": "Hold",
        "chain": [
            {
                "agent": "Market Analyst",
                "output_type": "report",
                "key_claim": "RSI=68, 接近超买区域",
                "confidence": "medium"
            },
            {
                "agent": "Fundamentals Analyst",
                "output_type": "report",
                "key_claim": "ROE 25%, 盈利能力优秀",
                "confidence": "high"
            },
            {
                "agent": "Bear Researcher",
                "output_type": "argument",
                "key_claim": "估值 PE=35, 高于行业均值 22",
                "evidence": "PE 35 vs industry 22",
                "convincing": true  ← 被 RM 引用
            },
            {
                "agent": "Research Manager",
                "output_type": "judgment",
                "decision": "Hold",
                "basis": [
                    "Bear's valuation concern is well-supported",
                    "Bull's growth thesis lacks short-term catalysts"
                ],
                "winning_side": "Bear"
            },
            // ... Trader, Risk Debates, PM ...
        ]
    }
    """

    def __init__(self, trace_id: str):
        self.trace_id = trace_id
        self.chain = []

    def record_analyst_report(
        self, agent: str, report: str, quick_llm
    ) -> None:
        """从分析报告中提取核心主张。"""
        # 使用 quick_llm 提取 1-2 句核心结论（$0.001/次）
        try:
            extract_prompt = f"""
            从以下分析报告中提取最重要的一个核心发现（1-2句话）。
            只返回发现本身，不要其他文字。
            
            报告: {report[-1500:]}
            """
            result = quick_llm.invoke(extract_prompt)
            key_claim = result.content if hasattr(result, 'content') else str(result)
        except Exception:
            key_claim = "[extraction failed]"

        self.chain.append({
            "agent": agent,
            "output_type": "report",
            "key_claim": key_claim.strip(),
            "timestamp": datetime.now().isoformat(),
        })

    def record_debate_argument(
        self,
        agent: str,        # "Bull" | "Bear"
        response: str,
        round_num: int,
        quick_llm,
    ) -> None:
        """记录辩论论点及其是否被后续裁判引用。"""
        try:
            extract_prompt = f"""
            从以下辩论发言中提取：
            1. 核心论点（1句话）
            2. 引用的具体证据（数字/数据）
            只返回 JSON: {{"claim": "...", "evidence": "..."}}
            
            发言: {response[-1000:]}
            """
            result = quick_llm.invoke(extract_prompt)
            content = result.content if hasattr(result, 'content') else str(result)
            parsed = json.loads(content)
        except Exception:
            parsed = {"claim": "[extraction failed]", "evidence": ""}

        self.chain.append({
            "agent": f"{agent} Researcher",
            "output_type": "argument",
            "round": round_num,
            "claim": parsed.get("claim", ""),
            "evidence": parsed.get("evidence", ""),
            "timestamp": datetime.now().isoformat(),
        })

    def record_judgment(
        self,
        agent: str,           # "Research Manager" | "Portfolio Manager"
        decision: str,
        winning_side: str,    # "Bull" | "Bear" | "Balanced"
        basis: list[str],
    ) -> None:
        """记录裁判决策。"""
        self.chain.append({
            "agent": agent,
            "output_type": "judgment",
            "decision": decision,
            "winning_side": winning_side,
            "basis": basis,
            "timestamp": datetime.now().isoformat(),
        })

    def to_dict(self) -> dict:
        """输出完整追踪记录。"""
        return {
            "trace_id": self.trace_id,
            "final_decision": self._extract_final(),
            "chain": self.chain,
            "summary": self._generate_summary(),
        }

    def _extract_final(self) -> str:
        """从追踪链中提取最终决策。"""
        judgments = [
            e for e in self.chain
            if e["output_type"] == "judgment"
        ]
        return judgments[-1]["decision"] if judgments else "unknown"

    def _generate_summary(self) -> str:
        """生成决策路径摘要。"""
        if not self.chain:
            return "Empty trace"

        steps = []
        for i, entry in enumerate(self.chain):
            agent = entry["agent"]
            if entry["output_type"] == "report":
                steps.append(f"{i+1}. {agent}: {entry['key_claim'][:80]}")
            elif entry["output_type"] == "argument":
                steps.append(f"{i+1}. {agent}(R{entry['round']}): {entry['claim'][:80]}")
            elif entry["output_type"] == "judgment":
                steps.append(
                    f"{i+1}. ★ {agent}: **{entry['decision']}** "
                    f"(favors {entry['winning_side']})"
                )

        return "\n".join(steps)
```

```python
    # 在图执行后、每个节点完成后，通过 LangGraph callback 记录

    # 最终保存

        trace_dir = Path(self.config["results_dir"]) / company_name / "traces"
        trace_dir.mkdir(parents=True, exist_ok=True)
```
