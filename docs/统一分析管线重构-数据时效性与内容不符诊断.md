### 🔴 P0-1：「内容不符」API 路径 industry 注入丢失

**严重度**：🔴 极高

**位置**：`tradingagents/api_server.py:analyze()` / `analyze_get()`

**现状**：

```python
# api_server.py
ticker = req.ticker or "000001"
trade_date = date.today().isoformat()
final_state, decision = graph.propagate(ticker, trade_date)
# 0 处调用 get_industry / validate_ticker
```

**影响**：
- 跨 ticker 一致性计划做的"跨1 注入契约"（`industry/injection_contract.py`）依赖 `IndustryFramework().lookup(industry)`，industry 缺失时 fallback 到空 → **API 路径下跨1 / 跨3 / 跨4 修复完全失效**
- 6 模板的 `report_skeleton` 字段已存在但 `propagate()` 入口**没有读取**该字段（见 3.6）

**触发场景**：
- 任何用户通过 `POST /analyze` 或 `GET /analyze` 请求分析
- 通信线缆股（永鼎股份）被 SaaS 框架分析
- 6 模板的骨架约束在 API 路径下完全旁路

**修复方向**：
```python
# api_server.py:analyze()
from tradingagents.dataflows.a_stock_data import validate_ticker, get_industry

if req.ticker:
    ok, company_name = validate_ticker(req.ticker)
    if not ok:
        raise HTTPException(400, detail=f"无效 ticker: {req.ticker}")
    industry = get_industry(req.ticker) or ""
else:
    industry = ""

# propagate 透传 industry（需要 propagate 签名扩展或用 self.industry）
self._last_industry = industry
```

```python
# trading_graph.py:propagate() 入口
# 读取模板的 report_skeleton，注入到 analyst prompt
from tradingagents.planner.template_matcher import TemplateMatcher
tm = TemplateMatcher()
best_template = tm.match(req.message, industry=industry)
if best_template and "report_skeleton" in best_template:
    skeleton = best_template["report_skeleton"]
    # 注入到 setup_graph 的初始 state 或 analyst prompt
```

### 🔴 P0-2：「数据时效性」`trade_date` 在 API 层被硬编码为今天

**严重度**：🔴 极高

**位置**：`tradingagents/api_server.py:analyze()`

**现状**：

```python
ticker = req.ticker or "000001"
trade_date = date.today().isoformat()  # ← 硬编码
final_state, decision = graph.propagate(ticker, trade_date)
```

**影响**：

1. **用户失去回测能力**：无法传历史 `trade_date` 字段（`AnalyzeRequest` schema 没有该字段）
2. **同一天多次请求的"幂等性"被破坏**：今天 09:00 跑一次、14:00 跑一次，两份报告的 `trade_date` 一样，但第二份应该看到盘中数据更新
3. **数据可能滞后**：`propagate(ticker, today)` 内部会拉取"今天"的数据，但 K 线、新闻可能在收盘后才更新；周末/节假日跑会取到周五/节前最后一天的数据

**触发场景**：
- 用户在 09:30 请求分析，报告说"2026-06-04 收盘数据"，但当天还在交易
- 用户在周日请求分析，报告 header 说"2026-06-04"（周日），内容里是 06-02 周五数据

**修复方向**：
```python
class AnalyzeRequest(BaseModel):
    user_id: str = "default"
    message: str
    ticker: str | None = None
    trade_date: str | None = None  # ← 新增 (YYYY-MM-DD)

# analyze() 内
trade_date = req.trade_date or date.today().isoformat()
```

### 🔴 P0-3：「内容不符」`GET /analyze` 默认 ticker 被静默替换为 `000001`

**严重度**：🔴 极高

**位置**：`tradingagents/api_server.py:145-147` + `analyze()`

**现状**：

```python
@app.get("/analyze", response_model=AnalyzeResponse)
async def analyze_get(user_id: str = "default", message: str = "", ticker: str = ""):
    req = AnalyzeRequest(user_id=user_id, message=message, ticker=ticker)
    return await analyze(req)

# analyze() 内：
ticker = req.ticker or "000001"  # ← 静默替换
```

**影响**：
- 用户 `GET /analyze?message=分析茅台` 不传 ticker
- 系统**静默**分析 000001（平安银行）并返回
- 用户看到的报告是平安银行，与"分析茅台"的意图**完全不符**

**触发场景**：
- 前端跳转链接不传 ticker
- 用户复制 URL 漏掉 `?ticker=600519`

**修复方向**：
```python
# analyze() 入口
if not req.ticker:
    raise HTTPException(400, detail="请提供 ticker（6 位 A 股代码）")
# 或者从 message LLM 提取 ticker（兜底）
```

### 🔴 P0-4：「内容不符」跨 ticker 一致性失效（字数极差 14,643 字）

**严重度**：🔴 极高

**位置**：跨 ticker 一致性计划的全部修复（`injection_contract.py` / `industry-fields-whitelist.md` / 6 模板 `report_skeleton`）**未在 propagate 路径生效**

**现状**：

| 指标 | 目标 | 实际 | 偏差 |
|------|-----:|-----:|-----:|
| 报告长度标准差 | ≤ 100 字 | **5,294 字** | 53× |
| 总字数极差 | — | 14,643 字 | — |
| Fundamentals 段极差 | ≤ 100 字 | 10,461 字 | 105× |
| News 段极差 | ≤ 100 字 | 10,152 字 | 102× |

**影响**：
- 跨 ticker 一致性计划（`docs/跨-ticker-consistency-cd9ee111`）的 4 项修复（跨1 注入契约、跨2 ReportRenderer、跨3 白名单、跨4 report_skeleton）**在重构后部分失效**：
  - 跨1：`injection_contract.py` 存在但 API 路径不传 industry → 失效
  - 跨2：`ReportRenderer` 已被官方判定为 bug 来源删除 → 失效（但有"列4/列5"修复收益）
  - 跨3：`industry-fields-whitelist.md` 存在但 `build_instrument_context()` 在 API 路径不被调用 → 失效
  - 跨4：6 模板 `report_skeleton` 字段已存在但 `propagate()` 不读取 → 失效

**触发场景**：
- 同一管线跑 6 只 A 股，6 报告字数差 2.83 倍
- 用户做横向对比时无法相信"同一管线 = 一致深度"

**修复方向**：

**短期（最小修复）**：
- `propagate()` 入口读取 6 模板的 `report_skeleton`，注入到 setup_graph 初始 state
- `build_report()` 末尾追加"章节字数统计"段，暴露极差（让用户看到不一致）

**中期（深度修复）**：
- analyst prompt 加 `max_length` 硬约束（如 `fundamentals_report` 8000-12000 字）
- 恢复 `ReportRenderer` 的字段格式化部分（不是正则解析部分）做"长度归一化"
- 在 propagate 内对每个 analyst 输出做 token 截断 + 警告

### 🟡 P-新 A：「内容不符」行业反模式约束"看不见"

**严重度**：🟡 中

**位置**：`tradingagents/industry/injection_contract.py`（注入位置）+ analyst 输出

**现状**：
- 6 份报告 grep "行业锚定|行业约束|禁止使用|反模式" = **0 提及**
- `injection_contract.normalize_injection()` 把 anti_patterns 注入到 analyst system prompt
- 报告**不显示**"行业锚定"段，用户看不到约束是否生效

**影响**：
- 跨 ticker 一致性修复的"可见性"问题：效果在 prompt 层面生效但用户看不到
- 即使 industry 注入工作（guping 路径），效果也"看不见"

**修复方向**：
- `build_report()` 顶部追加"行业锚定"段：
  ```markdown
  ## 🎯 行业分析框架
  
  **行业分类**: {industry}
  **分析框架**: {framework_name}
  **禁止使用术语**: {anti_patterns}
  **重点关注指标**: {correct_metrics}
  ```
- 让反模式约束在报告输出中可见，便于人工 review
