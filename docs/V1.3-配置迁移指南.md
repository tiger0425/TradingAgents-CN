# V1.3 配置迁移指南

> 从 V1.2 升级到 V1.3 需要了解以下配置变更。大部分修复为 always-on（无需配置），仅 3 项有独立开关。

---

## 新增配置键

V1.3 在 `default_config.py` 中新增了 3 个配置项：

### 配置键详解

| 键名 | 默认值 | 对应 FIX | 说明 |
|------|:------:|:--------:|------|
| `fan_out_enabled` | `true` | FIX-1 | 分析师并行执行。开启后 4 个分析师通过 LangGraph Send API 扇出-汇聚并行运行，延迟降低约 67%。关闭则回退串行模式。 |
| `enable_checkpoint` | `false` | FIX-3 | V1.2 动态图检查点。基于 task-based SQLite 的状态保存/恢复，API 崩溃后从最后成功节点恢复。默认关闭以兼容现有行为，生产环境建议开启。 |
| `enable_causal_trace` | `true` | FIX-10 | 因果链追踪日志。分析完成后在 `results_dir/{ticker}/traces/{date}.json` 生成决策路径 JSON，支持决策回溯。 |

### 无独立开关的修复

以下修复为 **always-on**（默认启用且无需配置）：

| FIX | 名称 | 说明 |
|:---:|------|------|
| FIX-2 | 辩论路由枚举化 | 用 `latest_speaker` 枚举替代 `startswith("Bull")` 字符串匹配 |
| FIX-4 | deep_llm fallback | deep_llm 调用失败时自动降级到 quick_llm |
| FIX-5 | 辩论深度提升 | `max_debate_rounds` 默认值从 1 提升至 2 |
| FIX-6 | KB 覆盖率时效加权 | 覆盖率计算加入指数衰减因子 |
| FIX-7 | 上下文窗口管理 | LLM 结构化摘要替代硬截断 |
| FIX-8 | 工具调用死循环检测 | 滑动窗口模式检测重复工具调用 |
| FIX-9 | 文件并发安全 | `filelock` 保护所有文件写入 |

### 按场景推荐配置

#### 生产环境（推荐）

```python
# 全部启用
fan_out_enabled = True
enable_checkpoint = True       # 建议启用崩溃恢复
enable_causal_trace = True
```

#### 开发/调试环境

```python
# 串行模式 + 无追踪，便于逐节点调试
fan_out_enabled = False        # 串行方便观察每个 Analyst 输出
enable_checkpoint = False
enable_causal_trace = False
```
