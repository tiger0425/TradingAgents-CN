# V1.3 配置迁移指南

> 从 V1.2 升级到 V1.3 需要新增以下配置项。所有新项均有默认值，不修改配置即可安全升级。

---

## 新增配置键

V1.3 引入了 10 个架构修复开关，全部集中在 `default_config.py` 的 `fix_*` 命名空间。每个开关控制一项独立的架构修复，可单独开启或关闭。

### 快速启用（推荐）

在 `default_config.py` 或用户配置中添加：

```python
# 全部开启（推荐生产环境）
fix_parallel_analysts = True
fix_enum_routing = True
fix_v12_checkpoint = False
fix_deep_llm_fallback = True
fix_debate_quality = True
fix_kb_decay = True
fix_context_compression = True
fix_tool_loop_detection = True
fix_concurrency_lock = True
fix_causal_trace = True
```

### 配置键详解

| 键名 | 默认值 | 对应 FIX | 说明 |
|------|:------:|:--------:|------|
| `fix_parallel_analysts` | `true` | FIX-1 | 分析师并行执行。开启后 4 个分析师通过 LangGraph Send API 扇出-汇聚并行运行，延迟降低约 67%。关闭则回退串行模式。 |
| `fix_enum_routing` | `true` | FIX-2 | 辩论路由枚举化。用 `DebateRole` 枚举替代 `startswith("Bull")` 字符串匹配，消除路由脆弱性。 |
| `fix_v12_checkpoint` | `false` | FIX-3 | V1.2 动态图检查点。基于 task-based SQLite 的状态保存/恢复，API 崩溃后从最后成功节点恢复。默认关闭以兼容现有行为，生产环境建议开启。 |
| `fix_deep_llm_fallback` | `true` | FIX-4 | deep_llm fallback。当 deep_llm 调用失败时自动降级到 quick_llm，避免单点故障导致分析中断。 |
| `fix_debate_quality` | `true` | FIX-5 | 辩论深度与质量度量。默认辩论轮次提升至 2 轮，新增质量评分（证据相关性、反驳力度、新信息贡献）辅助裁判裁决。 |
| `fix_kb_decay` | `true` | FIX-6 | KB 覆盖率时效加权。覆盖率计算加入时效衰减因子，STALE/EXPIRED 数据的权重降低，避免不新鲜数据造成的虚假安全感。 |
| `fix_context_compression` | `true` | FIX-7 | 上下文窗口管理升级。智能截断策略优先保留决策链核心节点，裁剪冗余日志和已消费工具返回，信息丢失率降低约 60%。 |
| `fix_tool_loop_detection` | `true` | FIX-8 | 工具调用死循环检测。滑动窗口（最近 10 次）+ 模式匹配检测循环，超阈值自动中断并触发降级路径。 |
| `fix_concurrency_lock` | `true` | FIX-9 | 文件并发安全。`filelock` 库保护所有文件写入（KB/缓存/日志），防止多线程/多进程并发写入导致数据损坏。 |
| `fix_causal_trace` | `true` | FIX-10 | 因果链追踪日志。每个 Agent 节点自动记录 (decision, basis, source) 三元组到 `causal_trace.jsonl`，支持决策回溯。 |

### 按场景推荐配置

#### 生产环境（推荐）

```python
# 全部开启，获取最大稳定性和可观测性
fix_parallel_analysts = True
fix_enum_routing = True
fix_v12_checkpoint = True      # 生产建议开启
fix_deep_llm_fallback = True
fix_debate_quality = True
fix_kb_decay = True
fix_context_compression = True
fix_tool_loop_detection = True
fix_concurrency_lock = True
fix_causal_trace = True
```

#### 开发/调试环境

```python
# 关闭并行和压缩以便逐节点调试
fix_parallel_analysts = False   # 串行方便观察每个 Analyst 输出
fix_enum_routing = True
fix_v12_checkpoint = False
fix_deep_llm_fallback = True
fix_debate_quality = True
fix_kb_decay = True
fix_context_compression = False  # 保留完整上下文供调试
fix_tool_loop_detection = True
fix_concurrency_lock = True
fix_causal_trace = True
```

#### 最低兼容（V1.2 行为）

```python
# 全部关闭 = 完全回退到 V1.2 行为
fix_parallel_analysts = False
fix_enum_routing = False
fix_v12_checkpoint = False
fix_deep_llm_fallback = False
fix_debate_quality = False
fix_kb_decay = False
fix_context_compression = False
fix_tool_loop_detection = False
fix_concurrency_lock = False
fix_causal_trace = False
```

## 已有配置变更

以下 V1.2 已有配置在 V1.3 中行为有调整：

| 键名 | 变更说明 |
|------|---------|
| `fan_out_enabled` | 与 `fix_parallel_analysts` 功能重叠，前者将被逐步废弃。建议新配置统一使用 `fix_parallel_analysts`。两者同时设置时 `fix_parallel_analysts` 优先。 |
| `enable_causal_trace` | 功能与 `fix_causal_trace` 一致，同样将被逐步废弃。 |
| `max_debate_rounds` | 当 `fix_debate_quality = true` 时，默认值从 1 提升至 2。用户自定义值不受影响。 |
