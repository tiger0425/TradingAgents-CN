## 六、架构文档中的 Agent 计数矛盾

### 冲突 6：Agent 总数在不同文档中不同

| 文档 | 表述 |
|------|------|
| `LangGraph辩论架构.md` L6 | "LangGraph StateGraph + **13 Agent** 多智能体协作" |
| `架构缺陷修复方案.md` | 分析师 4 + 辩论 2 + 管理 3 + 风控 3 = **12 Agent**（不含消息清除等辅助节点） |
| `开发进度表.md` 1.1 | "StateGraph + **15 节点**"（节点数，非 Agent 数） |

**根因**：如果计入 Social Media Analyst（它确实是一个独立 Agent），总数为 4 分析师 + 2 研究员 + 3 管理（Research Manager/Trader/Portfolio Manager）+ 3 风控 = **12 个 LLM Agent**。加上 ContextAssembler（非 LLM 节点），加上消息清除节点等，图中总共 15 个节点。"13 Agent" 可能是把 Macro Analyst（仅存在于晨会模板中）计入。

**建议**：统一为 "12 个 LLM Agent（4 分析师 + 2 辩论研究员 + 3 管理层 + 3 风控师），扩展模板下可达 13"，或者精确说明包含了哪个额外的 Agent。
